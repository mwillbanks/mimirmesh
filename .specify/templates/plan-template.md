## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- [ ] Live discovery gate: Engine-owned capabilities are sourced from live runtime discovery, not hard-coded tool lists.
- [ ] Upstream runtime gate: Containers run real upstream startup commands and config translation paths.
- [ ] Readiness gate: Required bootstrap/indexing steps are automated and verified before runtime is reported ready.
- [ ] Degraded truth gate: Failure states include proven root cause and impacted capabilities from live execution.
- [ ] Local-first gate: Local/private execution is selected when capable; any hosted fallback is explicit and justified.
- [ ] Monorepo boundary gate: Reusable logic stays in `packages/*`; `apps/*` contain runnable entry points only.
- [ ] Modularity gate: No oversized junk-drawer `index.ts`; routing, adapters, bootstrap, and diagnostics are separated by concern.
- [ ] CLI experience gate: CLI-facing changes use Pastel, Ink, and `@inkjs/ui`, show visible progress for long-running work, and preserve a shared state model/visual language across TUI and direct commands.
- [ ] Testing gate: Package-local tests cover package changes; root tests cover regression/workflow.
- [ ] Documentation gate: `docs/features/*` updates include observed behavior, config requirements, bootstrap steps, degraded modes, and validation results.
