# Failure modes

Watch for these failure modes and avoid them.

## Over-triggering

Activating on every short or slightly ambiguous message creates friction and wastes tokens.

## Clarification as a crutch

Do not use this skill to compensate for failing to read:

* the specification
* the approved plan
* repository rules
* execution-mode instructions
* prior user corrections

## Wrong target

Do not ask the user for decisions that belong to the manager.
Do not ask the manager for decisions already resolved by scope or instructions.

## Too many questions

More questions are not better. Each question must materially affect execution.

## Low-value questions

Do not ask questions whose answers would not change the work in a meaningful way.

## Endless loops

Only one bounded clarification round is allowed, plus at most one blocker-only follow-up.

## Indefinite waiting

Unattended mode must not wait indefinitely or re-open repeated time windows.

## Approval inflation

Do not require approval when it adds no value.

## Verbose telemetry

Do not retain verbose logs or narrative run history. The skill must not cost more to monitor than it saves through avoided rework.
